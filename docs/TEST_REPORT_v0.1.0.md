# AntPlot v0.1.0 Test Report / 测试报告

**Test date / 测试日期:** 2026-07-15  
**Repository / 仓库:** [plumageF/AntPlot](https://github.com/plumageF/AntPlot)  
**Test runner / 测试入口:** `tools/final_acceptance.py`  
**Regression runner / 回归入口:** `tools/recognition_regression.py`

## 1. Purpose / 测试目的

This report verifies the public AntPlot package against representative HFSS/VNA-style antenna data. It covers recognition, unit conversion, derived S11/Smith data, multi-curve overlays, project-band thresholds, operation modes, and backend preview/export parity.

本报告用于验证公开版 AntPlot 对典型 HFSS/VNA 天线数据的处理能力，覆盖数据识别、单位转换、S11 派生、Smith Chart、多曲线叠加、工作频段阈值、操作模式以及后端预览与正式导出一致性。

## 2. Test environment / 测试环境

- Windows PowerShell
- Python 3.10+ virtual environment
- NumPy and Matplotlib from `requirements.txt`
- Prebuilt React/Vite frontend in the portable package
- Metrics are calculated from raw data or explicitly recorded interpolation, never from image pixels

## 4. Recognition regression / 识别回归

The dedicated regression suite passed the following five edge cases:

- non-standard S11 column: `solution_data / rectangular / Freq / S11_dB`
- positive Return Loss: `ReturnLoss_dB` with sign-convention info
- negative Return Loss: `ReturnLoss_dB` with a caution warning
- far-field grid: `far_field / radiation_grid`, not a normal curve
- Theta/Phi angular family: `far_field / radiation_polar`, `RealizedGain`

## 5. Generated artifacts / 生成文件

The result archive contains the actual backend-generated files from the acceptance run, including:

- S11 PNG/PDF/SVG previews and exports
- multi-curve overlay PNG/PDF

- 410–490 MHz threshold-band PNG

## 6. Reproduction / 复现步骤

```powershell
Expand-Archive .\downloads\AntPlot-Windows-Portable-v0.1.0.zip -DestinationPath .\AntPlot
cd .\AntPlot
.\install_and_start.bat
```

For command-line regression after dependencies are installed:

```powershell
.\.venv\Scripts\python.exe tools\recognition_regression.py
.\.venv\Scripts\python.exe tools\final_acceptance.py
```

For UI verification, open `http://127.0.0.1:4173/`, import one of the CSV files from the data archive, confirm the mapping, generate the backend preview, and compare the result with the corresponding image in the results archive.

## 7. Known limitation / 已知限制

The legacy acceptance script still references a user-provided path `D:\CSV\11.7.csv`. That path is machine-specific and its current content is a Realized Gain far-field report. For a portable/public test, use `examples/s11_cases/case01_hfss_standard.csv` as the standard S11 fixture. This should be changed in a future test-runner cleanup so public acceptance does not depend on a local drive letter.
